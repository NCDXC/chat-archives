> On Feb 9, 2022, at 11:01 AM, KE1B via Chat <chat@ncdxc.org> wrote:
> 
> My current shack computer is a 2013-era 21” iMac, running the Yosemite O/S. While it works perfectly, it is getting a little “long in the tooth” and I am trying to migrate to a 2021 27” iMac (Intel processor). This is probably the last machine Apple will make with an Intel processor, and I wanted that because VMWare can’t create a Windows Virtual Machine on the newer M1 Apple silicon.
> 
> I am finding problems with getting my venerable IC-7600 to “talk” to the new computer under the latest OS, specifically through the internal USB interface in the 7600. On the older (existing) Mac, I don’t use the USB interface at all. I use a microHam microKeyer-II, with no problems (under both MacOS and Windows/N1MM), but the MKII cannot run on the latest Mac OS (Monterey) (no support, no driver). Oh the joys of upgrading OS’s.
> 
> I have tried all sorts of debugging tools, and am starting to think there may be something amiss with the USB port on the radio, and I just haven’t noticed it for years since I don’t normally use it. Does anyone have an Icom radio with the built-in USB port that I could borrow to test for a few hours? An IC-7600, IC-7610, IC-7300, or anything like that, that I could connect to the new Mac to see if it has the same connection problems as my old 7600? I do have an Elecraft K3S with a USB port, but since it uses a totally different driver, it’s not a good comparison to see whether that works or not. All of the Icom radios use the same SiLabs USB/UART bridge chip, so all use the same driver. Any “transmitting” that I would do would be at zero power into a dummy load. (I don’t have any output problems, and I want to eliminate any RFI issues.)
> 
> FYI: The problem I am seeing is, the radio does “connect” to the logging program (MacLoggerDX) using the Icom driver, but over a period of an hour or so, the connection drops. Using WSJT-X exacerbates the problem, causing fairly rapid disconnect, and loss of PTT control. Looking at the data being exchanged between the logger and the radio, there seems to be a lot of “junk” (unsolicited, improperly framed garbage) coming out of the radio. For any Icom experts, I have “transceive” turned OFF, so I shouldn’t be getting ANYTHING that isn’t solicited by a poll.
