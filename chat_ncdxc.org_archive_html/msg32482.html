As someone pointed out earlier for compound calls the FT8 protocol generates a hash value number and transmits that instead of the actual call sign. The hash function produces 10 bits so the possible number of possible hashed values is only 1024 (2^10 = 1024). Therefore the odds of 2 strings (call signs) hashing to the same number is 1 in 1024, in other words quite high.
