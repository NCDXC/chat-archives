Not acknowledging a report is an incomplete exchange (as required for validation).  It should not count and it's a stupid format to propagate.

I can't be any plainer.  What is submitted for validation (LOTW, DXCC) is a COMPLETED exchange.  Yes, I know that EME/MS folks live in the gray too.  That doesn't make it right.

However, knowing that the software variant being used now has limits and failings, I log and keep at it (when no RR73).  The standard was created for a reason; some apps ignore the standards and allow incomplete exchanges to be logged.

Some (Dom) also have the 'worked before' feature on so the software is deaf to your call, for the day (on that band).  That too is simply stupid.  It's more efficient to simply work, log again and move on (as contest stations do).

Good luck folks, the short duration of this trip and the wimpy signals (bad path with the mountain in our way, long path is not feasible) will make it an utter slug fest.  I was blessed, no other way to put it.  I know that.  Both contacts in 4 minutes of each other.

And yah, a big mouth here too (it's a 'feature' hi hi).  I'm still working on better ears (lower noise floor and low band RX array).

On 11/16/2023 2:21 PM, Clay Couger wrote:
There are some folks that run FTx without sending rr73 so they can work  more people in a pile up. The POTA folks even document this practice at https://docs.pota.app/docs/hunter_guide-english.html#ft8ft4-hints. I do not agree with not sending rr73 as it causes hounds to keep sending signal reports.

Don’t transmit on the same frequency as the activator! WSJT-X was designed to be used in split mode. Select the option “Hold Tx Freq” and select an open portion of your waterfall before calling an activator.

 

Tail-end an existing QSO. Anticipate when the current station being worked will send 73 and send your exchange in the same transmit window.

 

Tip: Skip sending your grid square when hunting an activator when using FT8/FT4.

 

Once you send 73 to an activator, you might not receive a 73 in return if the activator is working a pileup. If you received a signal report, you should log the contact.
