Dear Users,

I'm pleased to announce that WSJT-X Improved v2.8.0 is available for download. This is a GA release that contains all the “stable” developments that have been successfully tested and optimized in the various 2.7.1-devel versions, as well as several new features, bug fixes and other enhancements. The most recent include:

  • Fixed an inconsistency where with all prior WSJT-X and WSJT-X Improved releases, incorrect frequencies could be logged and posted to PSK Reporter when non-band/mode default frequencies had been selected.
  • Fixed a long-standing flaw where the third callsign in combined messages was sometimes displayed as <> after clearing the DX Call box by right-clicking on the DX Call button or pressing the Esc key. (The latter also applies to WSJT-X, by the way.)
  • All program features now work on Windows out of the box, i.e. without having to install OpenSSL manually. The latter is only necessary if another, incompatible OpenSSL version is already installed on the computer.
  • For EME operation, a new option has been added to the Astronomical Data window to enable Tx shift relative to an Rx sked frequency to enable wide in-band separation of Tx and Rx frequencies.
  • Q65 submodes 15A, 15B and 15C can now be decoded with EME delay, for experimental purposes. Additional submode buttons make operation easier. The first Q65-15A EME QSO took place on March 2 and went smoothly!
  • Better readability of the green “goal post” when Dark Style is switched on.
  • Improvements for Fox operators: Inadvertent transmissions on frequencies overlapping the WSPR sub-bands is now prevented. And the Fox operator is notified if there is probably another Fox station on the frequency.
  • Better support of SuperFox mode for Wait and Call: It is now possible to activate Wait & Call also in SuperHound mode, even if the SuperFox has not yet been received. However, “blind calling” is still prevented to keep the bands clean!
  • Added a new Quick Filter category "Ignore stations worked before on band".
  • Playing of the alert sounds is now muted for callsigns from the "Ignore list", or from the three "Ignore …​" Quick Filter categories, unless you have checked the BP (bypass) checkbox.
  • Numerous other code improvements.
  • Furthermore, the “improved” features are now also described in the User Guide (see sections 1.2 and 16 in particular).

For those upgrading from WSJT-X v2.7.0, I would also like to mention such great features as High DPI scaling (for 4K monitors), three different GUIs (WSJT-X layout, AL and widescreen), the Wait Features, optional band buttons, filters, audio alerts, Full Duplex Mode, TCI support, band hopping, switch to dark style with one click, AutoSeq also for JT65, and an option to Halt Tx when SWR > 2.5. For more information, read the User Guide or watch the video tutorials on my YouTube channel.

Everyone is therefore recommended to update to this new version!

As usual, you will find the downloadable files on my SourceForge page. Please note that they are now located on the “Files” tab in the new “WSJT-X_v2.8.0” folder.

Here are the direct links to download the most important packages:

  • wsjtx-2.8.0-win64_improved_PLUS_250314.exe   (Windows 64-bit, WSJT-X GUI)
  • wsjtx-2.8.0-win64_improved_AL_PLUS_250314.exe   (Windows 64-bit, AL GUI)
  • wsjtx-2.8.0-win64_improved_widescreen_PLUS_250314.exe   (Windows 64-bit, widescreen GUI)
  • wsjtx_2.8.0_improved_PLUS_250314_amd64.deb  (Linux, WSJT-X GUI)
  • wsjtx_2.8.0_improved_AL_PLUS_250314_amd64.deb  (Linux, AL GUI)
  • wsjtx-2.8.0_improved_PLUS_250314_Darwin.dmg  (macOS x86_64, WSJT-X GUI)
  • wsjtx-2.8.0_improved_PLUS_250314_M1.zip  (macOS M1, WSJT-X GUI)

Thanks to everyone who contributed to this great new WSJT-X Improved release!

Very important: If something doesn't work as you think it should, please check the User Guide first!

And I would also like to highlight these two documents again:

  • How_to_deal_with_rig_control_errors.pdf
  • Was_tun_bei_Rig_Control_Fehlern.pdf

Enjoy the new release!
