Paul, 


Which is the beauty of the The Bespoke OQRS.
When claiming contacts we ask the submitter to input their name and mailing address.


We don't depend on a PayPal address, some:

1. people never update that address and mail piece was returned

2. are in national language that we can't print, or mail sorters can't read


On an early DXpedition I used the PayPal address, it took Tim and I weeks to unravel that mess, lesson learned.

The process we use today seamlessly integrates the donation system with OQRS.

The donor inputs his/her address. We still get some errors because the user entered their own address incorrectly. 

Our OQRS process does the same, the user inputs their information, gets the QSL manager out of the middle.


All in all, the USPS return rate is small and I don't see USPS doing much to address that issue. Especially since the rejected mail piece was "supposed" to be handled manually and seemingly they aren't. And anyway, OCR isn't perfect, there will always be rejects.


In my experience the number of DXers sending an SAE and money to the QSL manager is declining. One reason could be our QSL manager is in the UK, higher postage vs. in country.

On the Clipperton project we received 190 letters from DXers. 


For example:

VP6D in 2018    555 letters

VP8PJ in 2020    301 letter

TX5S in 2024      190 letters


OQRS and LoTW have changed the game.

This is our experience, other QSL managers may be (are probably) different.

73,
GS K5GS

Yes Gene.  This is a common problem with letters to USA.  

The problem is with the USPS sorting machine.  
Sometimes it is unable to read the address format.  
It sticks a yellow sticker on with garbage saying it cannot forward. 

Ideally this mail that gets rejected should be hand sorted for correct routing but USPS just return it.  

If you remember way back we formatted OQRS to correctly address USA letters to avoid this problem but I’d say 1 out of 100 get returned to me still.  

It needs to be highlighted to someone with influence at USPS.  


Note:
Tim is referring to the OQRS software developed by: PDXG, Tim M0URX and Charles M0OXO.
