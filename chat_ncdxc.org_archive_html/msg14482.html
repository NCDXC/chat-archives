I had a similar delay problem 2-3 years back.  It was only with the lists run by Mailman (Chat, etc.).  It turned out the Mailman moderator/manager had my ISP (Verizon) on a list requiring a double acknowledgement or something like that.  It was due to a special technique Verizon was using.  Due to complaints the special double acknowledgement was turned off eliminating the 1+ hour delays.

FYI:  Interesting that this delay problem is appearing on the NCDXC reflector. It's been a problem on the UDXA reflector (SLC) for a long time, hours delay and more in some instances. Recently, it has been a problem for the MHDXA (Colorado) reflector.

The guys using the Denver area reflector managed to convince some of the ISPs, that there really was a problem, and it was not the customer's problem - which is the usual response. After some serious analysis it was reported that there were instances of ISPs (or whatever) where the incorrect data handling protocols were being used, leading to mail going nowhere until queues were emptied.

I don't know the details, but I think there is a solution. The usage of the SLC reflector has gone to zero, and an alternatvie - Slack - is being used. It's OK, but the format on the screen leaves something to be desired.
