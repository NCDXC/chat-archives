On Sun, Feb 19, 2023 at 2:48 PM Rick NK7I via Chat <chat@ncdxc.org> wrote:

Hi Al,

The VE station was most likely using a program other than WSJT-X (there are a few programs 'branches' based on the original code).

My guess was that s/he was using JTDX or MSHV, which allows multiple stations to be worked in 1-5 streams, SIMILAR to F/H mode but does not cause the tone shift at the calling station unless YOU are in F/H mode.  (There is no extra space in the data stream to add a 'bit' to mode hint, sadly.)

Or s/he could have been using F/H mode set to one stream (which hears your unshifted [NOT hound mode] tone in the response and work just fine for the final RR73).  Since the station sent on the 'even' cycle; there is no way to know, just to guess.

Confusing, right!?!?!

NOW you have one more clue to understand why some recent teams (not just Bouvet) have created concerns (problems) when using FT8.  There are too many unknown variable UNLESS the team/station announces what program and mode will be used (like Bouvet did) AND sticks to it.

The variant apps help to add to the confusion, mostly by breaking the foundation that the JT team built (txing as Fox only on even; and more).

The source code is in the public domain, the reasoning for the choices made varies among the branches.

I stick with the original line of WSJT-X, which is still being matured.

73,
Rick nk7i
