Hi Roberto,

 

It’s gotten confusing with DXpeditions using F/H, as well as MSHV. FWIW, the giveaway seems to be if you see successful calling stations being moved down onto one of the streaming frequencies of the DX, it’s almost certain that the CALLING station has set to F/H. It doesn’t guarantee that the DX is set to F/H.  F/H only uses one protocol for cycle timing- The Fox is always even.  In F/H the DX is ALWAYS on that cycle- MSHV allows use of either odd or even cycle for either DX and callers.

 

If a DX station on MSHV has matched cycle with that used for F/H and is transmitting even, they can contact calling stations set to FT8 Normal OR F/H (but callers must be >1000Hz if the DX is set to F/H). If the DX station is on MSHV and on the opposite cycle (like FT8WW at the beginning), and if you, the calling station, are set mistakenly to F/H, you’ll end on calling them while they’re transmitting, and you’ll never “hear” them..

 

Remember that on F/H, the software will not “see” any callers below 1000Hz. MSHV allows callers on any frequency. If the DX is definitely on MSHV, you may gain a tactical advantage (as Paul has noted) calling them below 1000Hz.

And hams say that digi-modes are no-brainers!
