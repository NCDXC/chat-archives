Keith, I'm pretty much in agreement with you. I use Win4Yaesu for rig control of my FTDX10. It has a really nice GUI which is well thought out. It interfaces with the HRD logbook module, and I can even run DM780 if I want. I did upgrade to the latest HRD, though, as there are some changes to the logbook that are worth having, which have added functionality, brought compliance to the latest ADIF, and have corrected some older issues.
For contest logging, I just use the N3FJP suite of contest loggers, for which I bought a lifetime license, and for a very reasonable cost, I might add. I found N1MM to be too heavy for me, as I'm not a frequent contester.

Roger

It might sound a little weird but I like HRD free logging program (don’t like anything else HRD). I use HRD logging last free version that Simon put together. All these years later, works pretty good for me. I only do about 1000 QSOs a year so don’t want to spend much or change to anything else. I pretty much tried most of every other logging program out there but I got used to the HRD logging user interface. It works good with DDUtil 3.0 for the Anan and Flex which allows you to use the logging program without all the weight of the rest of the HRD program modules. I guess I wasn’t alone in liking the HRD logging program and not much else from HRD since someone wrote software to make it interface without needing all other modules.
