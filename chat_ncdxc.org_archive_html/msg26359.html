OK, brain trust. I’m hoping someone can tell me what’s going on here.

I have WSJT-X running perfectly well on my Mac (OS 12 Monterey). All of the “standard” frequencies are properly defined in the Frequencies panel.

Now, let’s say I want to go to 15m FT8. OK, I set the mode to FT8 and select 21.074 from the frequency drop-down menu.

For about one second, WSJT-X shows 21.074 on its display, but then IMMEDIATELY changes it to 21.073.941, a slight offset of 59 Hz. Stranger though, is that the program is setting the radio frequency to 21.074.100, a 100 Hz offset in the OTHER DIRECTION. I can make QSOs just fine, either calling CQ or clicking on a received call, but I suspect the displayed audio offset (e.g., 1500 Hz) is different from what someone else is seeing, although we can communicate just fine.

This happens regardless of band or mode. If I select FT8 on 20m, instead of 14.074, WSJT-X displays 14.073.994, and the radio is set to 14.074.100.
Same for FT4, e.g. on 15m I select 21.140, WSJT-X jumps to 21.139.940 and sets the radio to 21.140.100. It’s always around 59-60 low on the WSJT display, and always 100 Hz high on the actual radio frequency. This offset is small enough so that it doesn’t really interfere with making QSOs (no one goes THAT close to the audio passband edges), but it seems weird. Also, the offset amount is consistently the same.

Radio is an Icom IC-7600, RIT is disabled, WSJT offset is set to “Fake it”. Other than the slight frequency shifts, everything works fine, transmit and receive.

Also, there is no frequency offset when using my regular logger on CW or SSB; the displayed frequency on the logger is exactly what the radio is set to, and they track perfectly.
