On Wed, Mar 18, 2026 at 9:59 AM Greg De Hoogh via Chat <chat@ncdxc.org> wrote:
I've also been using 3.0.0 without any problems. I just installed 3.1.0 and will see if that produces the problem. Reading the release notes, it seems the main thing 3.1.0 brings is support for FT2. It also adds support for two stations with non-standard or compound callsigns to work each other (e.g. N6PM/QRP can work DJ123XYZA). Since both stations need to be using 3.1.0, that feature isn't going to be too useful for now. It also adds the ability to turn off a8 decodes if you've enabled the multithreaded FT8 decoder (a8 decodes seem to be wrong much more often than they're correct).

In other words, if 3.1.0 is giving you problems, I recommend just going back to 3.0.0 for now.

Greg N6PM
