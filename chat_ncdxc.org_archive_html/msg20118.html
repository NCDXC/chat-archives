On Mon, May 25, 2020 at 10:58 PM Rick Bates, NK7I via Chat < chat@ncdxc.org> wrote:

If you're new to WSJT-X (or slightly used), here is a trick I've used during initial setup on a few different computers (since I access my station remotely).

WSJT-X keeps it's own log, even if you are using another logging program.  Consider it a backup log for WSJT-X modes.  The log is an ADI file with another file containing ALL the decodes you've seen (whew!)

When you're setting up WSJT-X (or want to update that log with your previous digital contacts), with WSJT-X NOT running;

1)  Import your WSJT-X log ADIF file into your main logging software (just in case);

2)  From your main logging program export the simplest ADIF file you can (typically using EQSL format is sufficient);

3)  Move that file into the WSJT-X folder;

4)  Replace the WSJT-X log file by renaming the WSJT-X log file, then;

5)  Rename the export you just made to the original WSJT-X file name.  Restart WSJT-X

WSJT-X ignores all the non-digital mode contacts so now the colors showing new grid, country, zone etc. are correct.  You only have to do this (again) when you're setting up another computer (installing WSJT-X).  Once synced, the logs should stay synced.

You can then use JTAlert OR WSJT-X to update your main long (not both) on each contact AND the WSJT-X log is accurate (for digital).

Also in that directory is a *all file.  You can use (search) that when there is a DX station telling you that they want confirmation but they're not in your log; because one of the bugs in WSJT-X is that it doesn't ALWAYS log everything it says it does (and some of the signal reports logged are not what the exchange said, pay attention when you click to add into your main log). 

Happy tweedling!
Rick NK7I

PS for the moment, I'm hearing of significant issues with the beta RC1 and RC2 releases; it's better to stay with the current release at this point unless you are up to a challenge.
