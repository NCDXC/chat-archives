Rick is correct. I have written several emails about this topic as well.

Refer to this thread: https://wsjtx.groups.io/g/main/topic/102745471#47848

Using the ft8code program in WSJT-x you can see the two callsigns hash to the exact same bits (1000111101):

ft8code.exe cq p4/w9ev cm98
    Message                               Decoded                             Err i3.n3
----------------------------------------------------------------------------------------------------
 1. CQ                                    CQ                                       0.0 Free text

Source-encoded message, 77 bits:
00000000000000000000000000000000000000000000000000000000000001000111101000000



ft8code.exe cq P97USI/P cm98
    Message                               Decoded                             Err i3.n3
----------------------------------------------------------------------------------------------------
 1. CQ                                    CQ                                       0.0 Free text

Source-encoded message, 77 bits:
00000000000000000000000000000000000000000000000000000000000001000111101000000
