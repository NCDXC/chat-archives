What I need is an optical character recognition program to transfer my 10 or so ARRL logs!  (  I only started computer logging of non-contests this year.  ).  I don't suppose such a thing exists?  

+ I'm not aware of anyone every being successful with OCR of hand-written logs.

+ DF3CB's "Fast Log Entry" (FLE) application reduces log entry time by requiring that you only enter information that has changed from one QSO to the next. Recording contest QSOs where you work a long string of stations on the same frequency and mode are thus reduced to entering a callsign, with a time update every 15 minutes of "log time" (so that your logged QSOs will be time-accurate enough to generate LoTW confirmations). FLE generates an ADIF file that you can import into your logging application.

<https://www.df3cb.com/fle/>

+ DXKeeper, the logging component of the free-ware DXLab Suite, can be optimized for recording "already-completed" QSOs:

<https://www.dxlabsuite.com/dxkeeper/Help/LogCompletedMain.htm>

+ DXKeeper will highlight any errors you make so that you can immediately correct them. You can export an ADIF file if you plan to use another logging application.

+ I've been meaning to develop a voice-driven front-end for DXKeeper so you can record QSOs from a paper logbook by simply reading the QSOs you've worked, but haven't yet gotten around to it.

+ The consensus among those who have spent lots of time recording QSOs from paper logbooks is that entry speed is accelerated by re-runs of 1950s sitcoms; "I Love Lucy" is evidently optimal.
