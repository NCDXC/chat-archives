You should always check the signal reports before it is sent to the logbook as well, they can be very wrong.  Something in the program gets scrambled at times.

In some cases, even if 'log' is clicked, it isn't sent to the logbook (HRD in my case) and a search of the WSJT-X logbook provided help when tracking down QSL complaints (from the DX).

That search ALSO showed that complete QSOs had EXACTLY the same time stamps (NOT possible), so be alert, it's not quite there yet.

I haven't installed the new release candidate yet, this is on the 'official' release.

On 5/13/2020 9:28 AM, Paul N6PSE via Chat wrote:
I have been using WSJT-X v2.1.2 for a day and a half. 

There are issues you should be aware of:

1) It does do a slightly better job at decoding week signals. 

2) There are issues with handshaking signal reports. After 3-4 back and forth signal reports, I often have to manually intervene and send a RR73. This is a new problem with this version apparently. 

While I like the better decoding, the poor handshaking is an issue and I wish that I could roll back to the previous version.
