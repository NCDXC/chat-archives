Please excuse me if this is a dupe or already well known (people keep asking me so I don’t think it is…)…

If you need Terrain Profile data for use with HF Terrain Assessment, you no longer have to go through the hassles of locating, downloading and processing the terrain datasets from NASA or USGS.

Instead, available on K6TU.NET at no cost (you just need to register for an account on the site – no charge, no credit card required), you can simply fill in a form with the location (decimal lat/lon to 6 decimal places) click on Generate and within 30 seconds you will have an email pointing you at ZIP file containing all 360 azimuth directions – about 280Kbytes compressed in a ZIP file.

I have the entire 1/3 arc second National Elevation Dataset loaded on the server and also cache any hits on the NASA SRTM dataset for all locations between latitude 60 South and 60 North.

Its drop dead easy!

The motivation for this automation came over last year’s Christmas vacation when I had four folks ask me for help generating the terrain data – its not hard but its always time consuming and if you don’t do it regularly, the re-learning curve is a repeated PITA.

After the first three were done and the fourth arrived, I said “never again” and set about automating the process.

Please help yourself, help your friends…  true friends don’t let friends do terrain profiles by hand!


K6TU.NET also has a set of tools to use HFTA to generate a full antenna model for your horizontal polarized antennas over the actual terrain around your QTH.  Using these antenna models together with the propagation predictions automated by the site, provides you with an optimum set of predictions.  These predictions are being used by a number of major contesters who are working on qualifying for the 2018 WRTC.

Anyway – no commercial, just wanted to make sure y’all know about the terrain profile generation capabilities in case you need to use HFTA.

Stu K6TU
