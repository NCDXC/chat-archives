I've been using WSJT-X v2.1.2 since December and have no significant issues. I run JTAlert in conjunction with WSJT-X. Either WSJT-X or JTAlert is set to prompt me to log the QSO (can't remember which has this setting), then JTAlert is set to automatically log to DXKeeper, DXKeeper set to automatically send new logged QSOs to LoTW. No issues whatsoever... Except I have noticed that once my system sends a 73 or RR73 it disables Tx. If the other station doesn't receive the 73 and retransmits their last message, I have to manually prompt WSJT-x to send my 73 message again. This can cause multiple attempts to log the same QSO, so I have to make sure not to log the same QSO on each repetition.

 

Brian NT6F

 

 

On Wednesday, May 13, 2020, 09:28:18 AM PDT, Paul N6PSE via Chat <chat@ncdxc.org> wrote:

 

 

I have been using WSJT-X v2.1.2 for a day and a half. 

 

There are issues you should be aware of:

 

1) It does do a slightly better job at decoding week signals. 

 

2) There are issues with handshaking signal reports. After 3-4 back and forth signal reports, I often have to manually intervene and send a RR73. This is a new problem with this version apparently. 

 

While I like the better decoding, the poor handshaking is an issue and I wish that I could roll back to the previous version. 

 

Paul N6PSE
