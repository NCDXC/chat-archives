That was a reading error.  The 6.8 version link was provided for those that couldn't (for hardware reasons) run 6.9 and needed a path backwards.

Download the latest version (the top link), install that and you should have your log restored.

Again, note the version and date of release on the top link; the file should be:  setupHRD69017153.exe for the 'maintenance release'.

I'm no longer a fan boi of HRD but I have been using it well over 20 years now, used to most of it's quirks.

So far, LOTW upload remembers which 'station location' you've set for default (a fix); but I still see other things not fixed (1 of 12, so far).

I've not tested other issues yet, like export ADIF (reports as 'truncated' by DX Keeper).

We expected, more.

If you are using HRD, an updated version is available to download (same place, note the version number and release date).

It is supposed (don't know for sure yet) fix many of the issues of the previous release.  [Temporary fix for the was delete the QRZ lookup cache and turn OFF the HRD Alert feature.]  I have a list of bugs I found, I'll work through them today to see if any/all have been fixed.  After the terse and rude response when reporting the flaws, nope, they didn't ask so it's my list.

HRD upport has NOT been supportive; it's also clear that their beta testing, was inadequate, even flawed.  After a ten YEAR wait for an upgrade, make your own conclusion.  [I keep DX Keeper and N1MM+ logs updated, in parallel, just in case.  Belt, suspenders, rope, harness and seat belt.]

So far it has been running continuously, so that's a bonus.  What was actually fixed or now broken, time will tell.

While I understand the (discussion of) migration from Windoze to linux (even agree with it); there are still MANY apps that are Windows only that I use daily; so a full migration won't happen.  Some of those apps are specific and far from free.  

I'll stay on Win 10 (with add on protections) as long as I can.  3 of the computers here (that still work fine) are not Win 11 capable and I prefer to have all OS the same (except the linux I run, mostly singular tasks on some Pi).  The real issue is app support, once they drop Win10.

So some good news and some bad; I'll post if the HRD team has resolved at least the larger issues.  Honestly, some were just stupid.
