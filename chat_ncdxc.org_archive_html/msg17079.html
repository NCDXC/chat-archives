One major issue that happens here is that the codec level setting for the SignaLink (mic; receive side) gets MAXed at the start of the beta WSJT-X.  When I reverted to the older version, whatever is causing this problem, stayed.  I did not use a new directory.

Decoding still works, but the decoder algorithm is being grossly overdriven (noted by the audio level display topping out and red even during breaks).

There have been no Windows updates in a couple weeks to change this and WSJT-X has been flawless until the install of the beta.

I use the HRD radio interface, so there is another level of complexity but starting WSJT-X now requires an audio level reset to the proper level.

This is repeatable, so something 'broke'.  It is not mode dependent.
