Folks,

I’ve made an update to K6TU.NET<http://K6TU.NET> that now allows you to override the default path selection when creating a Prediction.

The original Prediction uses the Smoothed algorithm in VOACAP to combine both Short and Long path results into one result set.

More sophisticated users (that would be me for one ;-) expressed the desire to be able to break short and long path predictions apart.  That is now coded and live on the site.

You will see the brief instruction regarding how to enable this on existing Predictions in the UPDATE NEWS section when you login to K6TU.NET<http://K6TU.NET>.

Right now this option is only available on the normal “Prediction” type that you create with New and Prediction.  I will be adding the Short/Long path option over time to the other prediction types where it makes sense.

Enjoy!
Stu K6TU
