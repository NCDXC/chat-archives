David,

when doing the online DXCC application the order of cards does not matter. Just stack the cards in the order entered and the card checker can take them from the top and work down the stack.

BTW, I was able to get the DXCC desk to accept emailed paperwork (I scanned the signed off paper and emailed it.). This has since been made official. However, this is only for ONLINE applications with field checking. Paper applications will not be processed for months. 

The last application I processed last week was finish by the ARRL DXCC Desk in less than a day. 

73, Andreas, N6NU



On May 9, 2020, at 6:43 PM, David Jaffe via Chat <chat@ncdxc.org> wrote:

Wondering if someone can help with this. . . 

I have started an Online DXCC Application but, due to various problems, I would like to reset it and start over.  I have not submitted the application yet.  Is there a way to do that?  If so, how?

David WD6T

P.S. If you're curious why I want to do that . .  I had an issue with the ADIF file that I imported, as it was missing DXCC entity fields for many of the QSLs.  As a consequence, only about half of the nearly 500 QSLs were accepted.   I then figured out a work-around for this problem (I imported the ADIF into a logger and then exported it again).  Unfortunately, this seems to have had the effect of randomizing the order of the QSLs (more specifically, I believe that what happened is that the repaired QSLs are showing up first, followed by the originally-accepted ones.)  Having spent a long time ordering the cards, this new ordering will add a lot of work to for me.  So I'm thinking that if there is some way to reset the Online application and start over, I could re-import the QSLs and hopefully get the original order back.

P.P.S. If you're even more curious as to why I had missing DXCC fields, it's because:  a) I didn't use an electronic logger until recently b) I entered paper QSLs into LOTW to see if they got hits, using the TQSL tool c) for those that didn't get LOTW hits, I want to submit the cards and d) the TQSL tool does not include the DXCC entity field.
