Yeah, I should have mentioned to avoid the overload. FWIW, HamAlert will actually shut down a trigger that causes too many hits -  ask me how I know, hi hi. But a few of things to look for:
1. Specify mode, band, as much as you can to narrow down the hits.
2. You can specify what time of day you want to get alerts so it’s not going off while your sleeping.
3. Specify a location for the spotter. You can do this by state as well as CQ zone. So I just specify CA. For the most part anything in the neighboring states will still likely get spotted by someone in CA, but this is narrower than CQ zone so it creates less hits.
4. When you load up an entity list from ClubLog you can edit the generated list to narrow things down a bit according to you priorities. It doesn’t automatically refresh (AFAIK) so you need to refresh it every once in a while. For the most part, if you’re new to DX this feature is probably overkill anyway. But once the ATNOs get harder, it’s great.
5. They recently added POTA as a category. I’ve managed a 50 state POTA award, but I did it with DC as a wildcard and would like to get RI for full coverage, so I have a trigger that will let me know when there’s a POTA specifically in RI (not a frequent thing hi hi).
6. It also supports a Telnet interface so you can even integrate it into you logging program if you have that feature.

It’s really great during a DXPedition, as you can set it up to look for whatever bands you haven’t gotten yet and it will alert you once somebody spots them on a wanted slot/mode.
