I will be operating maritime mobile for 30 days on an expeditionary cruise ship M/V Spirit of Enderby from New Zealand to McMurdo Station, Antarctica and return, beginning February 8.

Does anyone know of any logging software that will read and import latitude/longitude or grid square location from a GPS receiver so that I can have a seamless method of QTH position reporting for those I am able to contact?

>>>The DXKeeper and WinWarbler components of the freeware DXLab Suite can do that.

>>>DXKeeper, the logging component of DXLab, enables you to define a "my QTH" for each location from which you've made QSOs; a "my QTH" specifies a latitude, longitude, grid square, postal address, CQ zone, ITU zone IOTA tag, primary administrative subdivision (state, oblast, prefecture, etc.), and secondary administrative subdivision (county, district, etc.). You specify a "my QTH ID" for each of your "my QTHs" (typically a town name, but you could instead use a grid square); each logged QSO specifies a "my QTH ID".

>>>WinWarbler, a component of DXLab that provides automation for making and logging CW, Phone, and digital mode QSOs, can connect to an NMEA-compliant GPS via a serial port, and automatically create a "my QTH" based on the current position, incorporating the current position's grid square in the "my QTH ID". WinWarbler can log CW, Phone, and digital mode QSOs directly to DXKeeper.

>>>See

<http://www.dxlabsuite.com/winwarbler/Help/PositionVector.htm>
