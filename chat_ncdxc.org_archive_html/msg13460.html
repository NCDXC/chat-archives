George –

 

Exactly the same with DXKeeper.  Log a QSO and CL is immediately (behind the scene) updated.  I upload the Q to LoTW on occasion usually putting the Q in the LoTW queue or doing a single upload – one key stroke.

 

The only manual portion of the current configuration is when LoTW comes back with confirmations updating DXK having pushed the SYNC LoTW button.  I have to click Update Club Log for each just updated record - that is it.  DXK displays the records updated by LoTW confirmations in list.  Then right-click and click on Update CL for each record.  The only manual operation in the conventional scheme.

 

With this new CL/LoTW it looks to me like the process is essentially reversed.  I see the same amount of automatic operation and manual operation just in the reverse direction.

 

The hole in the bucket for the new CL/LoTW features is one still has to update one’s master log with the newly update records on CL, those changed by LoTW confirmations just received.

 

73,

You guys are ahead of me because I haven’t actually tried it yet, but it “seems” like a very nice upgrade.

 

DXKeeper allows automatic uploads to ClubLog every time a QSO is logged in DXKeeper, if you configure it to do so.  But, there is, by design, no way to automatically upload to LotW.  (Dave says that would be too much of a load on LotW.)

 

However, with this new ClubLog-LotW feature, ClubLog can automatically upload to LotW, so it should no longer be necessary to manually upload from DXKeeper to LotW.  If I understand it correctly, this is a nice benefit.
