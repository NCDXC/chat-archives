Probably MSHV Al.  It allows you to use the one stream for 2 contacts at the same time without using f/h mode.  (The stock code sends the data twice; both MSHV and F/H break that down to one pass per stream.)

MSHV has other features like queue, a holding place for those calling, until they can be called back; choosing the number of streams and more.  That is the foundation for robot mode, call CQ and walk away as some do.  When the queue is empty, it calls CQ again.  (It does hang now and then, so walking away is not the best plan.)

WSJT-X treats it as F/H (without changing tones), even if it's not; so it didn't send a final RR73.  When MSHV sends the RR73, it is logged.  It doesn't wait for your RR73 so it can move along faster (also like F/H).

I use it sometimes when I'm pointed at AS for a new slot; knowing that the JA hordes will descend upon me.  In this way I don't have to be rude or ignore them but (AFTER I've worked the wanted DX) can enable that mode then work them until they get bored (20-30 callers later, typical).
