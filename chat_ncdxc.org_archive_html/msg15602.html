I had 2 dups, both FT8 and both a minute apart. On one of them I never received the RR73*, but I logged it anyway (recommended). The other seemed perfectly normal from my end.

There may be a logging problem in the current release of wsjt-x, but twice is much better than none. A patch for the problem would be to only make one log entry for each 2 minute period, but this is kinda hokey. I do know in normal FT8 mode, I can get two log dialogs if the remote end repeats his RR73.

The need for an ACK for ACK gets into the Communicating Generals problem in computer science. This problem is generally considered unsolvable. My general take is that if you have received all the expected exchange you log the QSO and let ClubLog/LotW sort it out.
