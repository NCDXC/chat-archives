WSJT-X "invented" multi-streaming with the advent of the F/H protocol prior to the Baker expedition.

Later, the MSHV developer chose to disconnect multi-streaming from the other requirements of F/H. Namely hound callers must be above 1000 Hz and the hound must move down to the frequency they where called by the fox to complete the QSO.

One can argue what is a good or bad design decision, but there was no innovation in MSHV with regards to protocol in this case.
As to the other automation "features" of MSHV I won't comment other than to observe the ARRL responded (to MSHV and others) with a DXCC rule change which requires a human in the loop.

Recent versions of WSJT-X such as used by TX5S no longer force the hound to be in hound mode. The fox will complete the QSO on the hounds calling frequency or move them to the calling frequency if the caller is in fact in hound mode. This improved compatibility with code forks (JTDX, MSHV, etc) as well as improved QSO close rates with folks who forget to check the hound box. Staying above 1000 Hz is still mandatory when calling a fox using WSJT-X.

TX5S saw significant QSO rate improvements due to this change as well as some key bug fixes that allowed up to 10 stations to be consistently called on each cycle if conditions supported it. It was not unusual to see a 400+ QSO hour which is really quite amazing.

Perhaps we should have done a better job of communicating the advantages of this particular WSJT-X release, but there has also been a general unwillingness to use WSJT-X by some teams. Perhaps now that will change with what we see unfolding with FT4GL. For expeditions, stability, reliability and consistency in software behavior should be the priority now that WSJT-X in fact outperforms any other derivative version in QSO rate.

Looking forward, Super Fox is a completely different protocol and while it is going to offer an improvement in S/N capability it will no doubt introduce a whole new level of confusion (and bugs) since it will not be backward compatible with any older versions of WSJT-X or derivatives. Let the fun begin!
