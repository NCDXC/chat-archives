On Dec 4, 2013, at 5:24 PM, Kevin Rowett <kevin@rowett.org> wrote: 

I'm pretty sure this clock uses WWVB on 60kHz carrier, which is still run by NIT.

In Nor Cal, it's only around 3AM that you'll get enough carrier to get the time set correctly for this clock…it may take up to a week to get it to be accurate.

It's also very slow to update around DST change.

If you like reading the TOD from your PC, then you might look at a task bar clock - "1st clock".  It does use NTP (as does most PCs these days), and that can be very closely synched to UTC.
