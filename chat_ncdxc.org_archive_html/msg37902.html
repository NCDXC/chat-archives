FWIW: I worked him using normal (non-F/H) FT8 at 06:10 UTC, and as of 07:04 he's still on 80m at -12 in Nevada County (dipole).

   Because of the compound callsign, many of his FT8 messages use a hash code instead of the literal callsign (it's part of the FT8 protocol). These messages are denoted by the < > or <...> markers. If you wait long enough - and you're otherwise getting good decodes - WSJT-X will eventually receive a literal callsign, calculate the hash code locally, and thereafter will know how to display the correct call. You can help speed up that discovery process by typing his callsign into the "DX Call" box in WSJT-X.

   - Richard,  NC6RJ
