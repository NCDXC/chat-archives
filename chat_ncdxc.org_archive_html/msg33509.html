I wanted to clarify and correct this.
There are no new features.
The are however many bug fixes in the Fox code. These bugs have existed since the original implementation of Fox/Hound mode in 2016.
Brian Moran, N9ADG, deserves huge thanks for making these much needed fixes. Rob, N7QT, worked with Brian to make this custom version available to TX5S.
The improvement in QSO's/hour throughput is significant. We estimate at least a 20-25% throughput increase for a given number of streams.
Brian created a custom build of the code, but it is hoped the fixes will be incorporated into the main code repository in the near future.
At this point the build has made nearly 100,000 QSO's, so it is very thoroughly tested and stable.
H44WA (Solomons) was the first to use this build. H40WA (Temotu) will also use it.
There are *NO* changes that would impact any other WSJT-X function.

I want to share an observation with you guys.   While on Clipperton, we used a very new version of WSJT-X that was optimized for DXpeditions. 

We were operating as a Fox. We saw that we could easily work Hounds AND those callers calling FT8 normal above 1000 Hz. Our WSJT-X did not require that their Xmit frequency move down into the 300-500Hz range. We did notice that Hounds seemed to be worked quickly and easily and non-Hounds often called for a long time.  That might be due to the fact that they were calling in very congested areas. We clearly saw that we worked guys calling above 1000 Hz that were never moved down below 1000 Hz. 

So some new changes/advances to WSJT-X. 

We will use WSJT-X  Fox/Hound on our upcoming H40WA trip in a few weeks.
