Looking at the key posted elsewhere, it is short and easy to crack. Using much longer keys would make it harder to crack the keys (and no, this doesn’t require sending more bits over the air). However, this would not eliminate the problem.

 

Ideally, the WSJT-X team would move to an asymmetric signature (LOTW uses something similar where it’s overkill; WSJT-X should implement it as well. And again, this doesn’t require sending more bits over the air). Yes, managing public/private keys is an extra complication; but it’s necessary to secure the system. And signing rare DX transmissions is a great feature, when it finally works properly.
