Actually, we do know the total volume of QSOs recorded in each system (they publicize that a lot), and they are all comparable. As of this morning, each system contains:

   LoTW: 470 million QSOs

If eQSL can process uploads in *seconds*, and LoTW takes *hours* (or days), and the difference between the number of QSOs processed is only 1.6:1, then there are systemic differences in the database processing that is causing slow performance in LoTW.
