Well, yes and no.  

The Heart Bleed bug has been in circulation since late 2011, but wasn't a problem for systems running older versions of SSL.  Only the very latest SSL version was affected.  Human nature being what it is, many users hadn't implemented the latest version until recently.  

While the bug has been out there for a couple of years, the SSL patch wasn't released until April 7th of this year.  Prior to that, the only work-around was a stop gap measure - recompiling system code to de-activate a key handshake protocol ("heart beat").  

Many users didn't adopt the work-around, and didn't feel pressure to address the problem until it hit the presses. This is how the Bit Coin site "Mount Gox" was hacked.  Who knows, may have been at play in the Target debacle as well? 

Now everybody is scrambling to implement the patch.
