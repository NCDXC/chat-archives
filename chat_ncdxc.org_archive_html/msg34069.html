On Wed, Feb 28, 2024 at 5:45 PM Bob Wilson, N6TV via Chat <chat@ncdxc.org> wrote:
Problem resolved.

I spoke with N7QT by phone and connected to their PC via TeamViewer (thank you, StarLink), and we were able to reload the K3 firmware using an external FTDI USB-to-Serial adapter and the RJ45 to DE-9F adapter cable (Elecraft P/N E980297) instead of using the K3 USB connector.

The recovery procedures for MCU LD issues are well documented in the K3 Utility help text, and they do work.  The key is to avoid using the K3 USB cable for firmware uploads.  

73,
Bob, N6TV

On Wed, Feb 28, 2024 at 16:29 Rick NK7I <rick.nk7i@gmail.com> wrote:
The state of MCU LD means it needs a firmware reload; that the firmware got corrupted.

Make SURE that the serial port is connected DIRECTLY to the K3 ONLY, not via anything else (P3, other hardware; DIRECT).

Then reload the firmware with the utility and when done, it should reboot and come back to life.
