AFAIK, there is no such graphic user interface for the K3 Configuration
Menu.  It would certainly be a useful applet.

There is a subset of your idea for one critical section of the menu.  That
is setting all the roofing filter parameters.  There are five or six
parameters for each filter and with the Sub-Receiver, there could be as many
as 10 filters.  That amounts to 50 parameters set manually using the VFO
knobs.  The K3 Utility software running on a PC or Mac provides a wonderful
way to set all the filters on in one window.  It is a checkbox layout of all
the filters vs. their 5-6 parameters, along with notes on restrictions, etc.
It is then trivial to reliably setup all the filters.  Extending that to all
the Configuration parameters would be welcome by many, I'm sure.

There is the ability to Save/Restore the entire K3 Configuration to a file
on your computer.  The file name is automatically formatted as a date/time
code along with the serial number of the unit.  I often save my
configuration for a given operation, contest or DXpedition so I can use it
as a starting point for a future similar event.  However, these files can
only be loaded into the same K3 with the same serial number.  This is
because several of the parameters are specific to a given unit.  Also,
another K3 probably won't have the same options so the Configuration file is
unusable, although that could be dealt with in the application.
Nonetheless, it is a very important feature for K3s that are shared by
different users, like guest ops or joint owners of a shared station or a
borrowed radio or a DXpedition kit, etc.

In addition to your suggestion of a better user interface, I'd like to see a
way to restore a K3 Configuration file to a different K3 by ignoring the
unit-specific parameters.  I've often had the situation where I want to copy
a configuration to one of my other K3s or to someone else's K3.  Now, that
has to be laboriously done by placing the radios side-by-side and scrolling
through both Configuration Menus with the VFO knobs.  I also think it would
be useful to have some starter configuration files for the various uses the
K3 is being put to.  That would go a long way in helping a new user get more
value out of their K3 rather than be frustrated trying to get it to work for
their application.
