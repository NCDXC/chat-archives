SuperFox works extremely well when signals are strong AND the Hounds have the right software. 

The biggest problem is getting people to install the latest release. Either -rc7 or -rc8 is required for SuperFox; many people complain that -rc6 doesn’t work, and they are right; it uses a now-abandoned authentication scheme.

Most DXpeditions have learned to start off using SuperFox, and stay that way as long as the rate is high. SF can support as many as NINE simultaneous streams. Once the pileups die down, it becomes more effective to run 2-3 streams of ordinary F/H. Since ordinary F/H uses only 50 Hz of bandwidth for both the Fox and Hound, the Fox can be decoded at a lower S/N ratio, typically down to -24 or so. SuperFox uses 1500 Hz for the Fox, so the noise bandwidth is increased 30-fold, making it harder to decode when weak.

The advantages of SuperFox are: 

(1) Very high rate possible relative to ordinary FT8 or F/H (which is important for big pileup DXpeditions)

(2) The software will not even allow a Hound to call the SuperFox unless it gets a decode. That eliminates all of the “blind callers” so that every station heard by the SuperFox is in theory workable. (This element is NOT insignificant.)

(3) SuperFox QSOs can be authenticated by the Hound, either in real-time or any time after the fact, as long as the Hound has Internet access at some point. The Fox does not need Internet access at all (important when the Fox is somewhere very remote, or with marginal access). 

Pirates can still operate using the DXpedition’s callsign, but they will NOT be authenticated, either in real-time or later, as long as the DXpeditioner keeps the private SuperFox key private. SuperFox keys are known only to the DXpeditioner, the owner of the authentication server, and myself (who issues the keys, and communicates them securely to the other parties). To date, no one has been able to “crack” the SuperFox authentication system, even though the algorithm is open-source. The private key cannot be inferred by the SuperFox’s transmissions, and brute force “guessing” is almost impossible. (The key comprises 16 alphanumeric characters, so that’s 36^^16 possibilities, about 8 trillion trillion combinations.)
