Below is a bug report I sent to ARRL’s LoTW support. Has anyone else noted this problem? It basically declares an error whenever the “Operator” field is not the same as the callsign certificate being used, which makes the whole thing pretty useless for multi-op contests or club stations. Perhaps it’s a Macintosh-only bug. I reverted to the previous version of the tQSL software (early May, 2020) and everything works fine with that. The example below is from the recent RAC Canada Day contest, where I used the K6MMM callsign on-the-air, but the operator was KE1B.

Using the latest version of tQSL for the Macintosh, I am getting errors on EVERY QSO where the “Operator” parameter is not the same as the callsign certificate being used. This never happened with earlier versions, and makes tQSL/LoTW completely useless for club stations (where the operator is NEVER the same as the club callsign).
