I’ve been using JTDX lately, but have not seen any noticeable difference in the decode levels.  I haven’t run any WSJT-X vs JTDX comparisons, either.

 

JTDX does have a ‘SWL’ decode feature that provides a multi-pass, multi-threaded analysis of the received data.   I turned it on and saw the CPU % go way up for JTDX, so turned it off again.

 

I read somewhere that -24db was the lower limit in WSJT-X for decodes and I have not seen anything lower being decoded by either program.

 

JTDX is here:  https://www.jtdx.tech/en/
