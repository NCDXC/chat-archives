Same here.  I find that DXLabs Suite has way too much functionality that I will never use and what I do use is very non intuitive.  I have learned how to use the small subset that I actually need, but I have to stay in practice or I will forget how it works.  It's nice that it is free and it is nice that Dave supports it, but it frankly NEEDS a lot of support.  I would rather have a slimmed down logger that doesn't need a lot of support.  

An example of the quirkyness is that if I want to upload to LOTW, I set the "since" date to only show QSO's I haven't already uploaded.  Maybe there is a way to get the program to automatically do this, but I haven't discovered it.  Anyway, when I do this, it always asks:  You have set a limited time span filter in place.  Would you like to remove that so that ALL QSO's will be loaded?  Then I have to answer NO every time.  It would be great if I could check "don't ask me again".  There are probably a half dozen weird characteristics of DX Lab just in terms of uploading to LOTW.  Occasionally, I have uploaded a contest log twice, and had duplicate QSO's.  I have to delete BOTH sets of QSO's from the contest and then reupload the contest ADIF again.  And deleting doesn't work by simply selecting all the QSO's and then hitting delete.  I forgot what the trick was for that.  Maybe hit ctrl-delete?  The program is just full of gotchas like that.

Will,

 

One firm LOUD suggestion:  Do NOT believe that DX Labs is a quick set up.  It is INTENSE; there are MANY MANY MANY settings in EACH program.  It can easily consume a week of time, getting it all dialed in the way you like it.

 

It does a LOT; but it is TEDIOUS to get set up.

 

And you WILL consume a lot of screen space (at first, not everything HAS to be up at the same time; but the apps DO interact so you'll run them too).  Some of the settings are not in your face; the learning curve is rather steep.

 

And one WARNING:  When DX Commander (rig control) starts up; it does not ask what the rig power out setting is; it TELLS the rig what it shall be.  I found out the hard (expensive) way; the amp was in OPER mode (the K4 set to 30 watts of drive) when I brought up Comander (which set the output to 100 watts).  I was operating remotely and didn't catch this on the rig display (or Commander).

 

The amp did not love the 3X drive power and the LDMOS quit.  That was a $1,000 lesson with shipping and insurance (the hard way).  That also limited me to 500 watts for Bouvet (the backup amp at that time).

 

If there is a way for Commander to ask what settings (including panadapter); I've not found them yet.

 

73,

Rick nk7i
