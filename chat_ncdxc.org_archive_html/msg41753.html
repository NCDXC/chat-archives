That raises one minor point about SuperFox. Unlike ordinary FT8, when the SuperFox sends a CQ message (not the normal exchange), the encoding allows for non-hashed transmission of callsigns up to eleven characters, including slashes, numbers, etc. (It’s a complex 58-bit field encoding a callsign in base-38. 2^58 = 2.9 x 10^17, and 38^11 = 2.4 x 10^17, which fits into the available code space. Geek mode off.)

The software is normally set to send a CQ every few minutes (even if they are running lots of QSOs), so that hounds can get the full callsign at least once in a while. This is necessary because in SuperFox mode, you CANNOT simply enter the station’s full call in the appropriate box and send the standard messages. WSJT prevents you from transmitting to a SuperFox unless you decode the Fox at least once. This prevents people calling who cannot hear the DXpedition, just adding more QRM.
