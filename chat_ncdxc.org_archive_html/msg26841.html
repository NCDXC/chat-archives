It's potentially 'simple' to transfer a dish.  If Starlink balks are just doing it in one request, I would suggest doing it in stages since transfer to a new party is against the original user agreement.  It's unrealistic to demand no transfers, folks change homes, change jobs... it's an issue they have to address better.

Moving a dish is now allowed ($25/month, turn that feature on, move it, turn that added feature off unless you want to keep using it in an RV for example).  Operation while in motion is not (FCC licensing limitation); but it does work (YMMV).

"Update" the billing name and address on the account (I've updated the name, so it's possible via tech support) and updated credit card information can follow.

I'm just happy to have a more reliable service here is rural Idaho.  There is ONE fiber cable for most of the other providers (costing a LOT more) and it gets cut 2-3 times a year (which kills cell service too) because of the rapid construction occurring 75 miles south (Coeur D'Alene).

Yes, the speed varies, my trees get in the way so I have occasional short dropouts but is generally 20 Mbps or faster, enough for my needs, enough for 4K video each up/down and audio. (I'll move it to the roof before snow now that I know it can manage the snow)...  The DSL network resets nightly, screwing some things up (IP address changing)...

One last thing, Starlink DOES NOT ALLOW INCOMING IP requests, so running any service on your network (web page, streaming audio/video apps), will require a VPN (added costs) as a workaround.

There was an unannounced test period last spring when they did allow incoming (testing IPv6?) and for that week, it was awesome (didn't last).  I'm hopeful they'll allow that again but with the limited IPv4 addressing they 'own', it's not possible now.
