FT8 uses fixed message formats and bit allocations. “Non-standard” call are sent as up to 11 characters, the other call sign is mapped as a hash code of 12 bits. Hash codes are note unique. FT8 displays the most recent <CALL> or if not in memory from a previous use.

This is why I think there are weird <…> calls sometimes.
