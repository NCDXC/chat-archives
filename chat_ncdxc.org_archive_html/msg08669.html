On Dec 10, 2014, at 10:36 AM, Rick WA6NHC via Chat <chat@ncdxc.org> wrote:

One side of the coin: it's stable, we don't need software users to much with it so hard code it.
Other side:  It changed, now WE have to put out an update to keep the users happy.

NEVER (and I mean NEVER EVER) rely on any interface you don’t own or that the provider hasn’t committed to maintain.

Hard code such data and sadly, you get to pay the price.

If you don’t want customers to mess with it, then include an update capability in the software so you can push updates to address such issues.

But - if you a scraping a web page, then what will you do when the format changes?  Better to provide your own (managed) source of the data.

I write software (a lot) and have for a very long time.  I am lazy - I much prefer to get the computer to do things without whining than do them myself.

But I am not a lazy programmer.

It’s NOAA’s web site and their prerogative to change it as they see fit.
